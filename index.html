Under the hood

How CUBED works

Three.js and WebGL

Three.js r128 translates the scene into WebGL drawing commands. A perspective camera with a 38-degree field of view looks toward the origin, while custom spherical camera controls provide orbit, wheel zoom, touch drag, and pinch zoom without OrbitControls or any add-on library.

A cube made from 27 cubies

The puzzle is not a painted box. It is a 3×3×3 collection of 27 independent cube meshes, spaced slightly apart to create the familiar dark grid. Each cubie receives sticker materials only on faces that point outward in the solved state.

Runtime-painted sticker textures

Every sticker is painted into a 256×256 canvas at runtime: a near-black surround, rounded colored inset, diagonal color shading, and a soft top sheen. Those canvases become Three.js textures, so the app needs no external image files. Choosing a color palette repaints the six shared sticker textures in place, preserving every cubie’s current position and orientation.

Face turns, pivots, and exact snapping

For each move, the affected cubies are temporarily attached to an invisible pivot group at the cube center. Rotating that pivot turns a face, middle slice, or the whole cube as one rigid unit. When the animation ends, cubies return to the main group and both position and orientation are snapped to exact quarter-turn increments. Quaternion-based orientation is converted and rounded so floating-point drift never accumulates.

Animation and the move queue

requestAnimationFrame drives every turn with an ease-out cubic curve. A FIFO move queue guarantees that buttons, keyboard moves, guide algorithms, scrambles, patterns, and automatic solutions never overlap. Half turns are one smooth 180-degree movement rather than two visible quarter-turn snaps.

Always-available solution engine

Every completed face, slice, and whole-cube turn is recorded in a reversible move ledger. SOLVE cancels unplayed queued work, safely lets a turn already in motion finish, inverts the completed move history, simplifies adjacent turns, and animates the exact return path to the solved state. Because every state in CUBED is produced by legal moves through this engine, the inverse sequence is an exact solver for any state the app can create.

Lighting and atmosphere

A hemisphere light, key directional light, colored rim light, and low ambient light reveal the cube’s form. The key light uses PCF soft shadow mapping. A points-based starfield slowly rotates behind the scene, a shadow plane catches the cube’s soft silhouette, and the Synthwave theme fades in a large perspective grid.

Smooth theme interpolation

Theme changes update CSS custom properties for the glass interface and also interpolate every scene parameter over about one second: light colors, intensities, star opacity, grid opacity, and ground-shadow strength.

No-framework interface

The entire interface is ordinary HTML, CSS, and JavaScript. CSS custom properties drive accents, backdrop-filter creates glass surfaces, Flexbox and Grid build the controls, and media queries transform the guide into a mobile bottom sheet. Chakra Petch and Sora are delivered by Google Fonts with system fallbacks.

Input and hosting

Pointer Events unify mouse, pen, and touch input. Wheel and two-pointer distance tracking handle zoom. Everything lives in this one HTML file, ready for GitHub Pages, with Three.js delivered from cdnjs.

How to use CUBED

Every control, from first turn to solved cube

Orbit, zoom, and automatic spin

Drag anywhere on the cube stage to orbit around the puzzle. Use a mouse wheel to zoom, or pinch with two fingers on a touchscreen. The Spin slider continuously orbits the camera: negative values rotate counter-clockwise, positive values rotate clockwise, and 0 turns spin off. Auto-spin pauses while you are dragging and continues during turns, algorithms, scrambles, and pattern playback.

Face-turn buttons and notation

The twelve move buttons turn the six faces clockwise or counter-clockwise: U D L R F B. Read a plain letter as clockwise while looking directly at that face. A prime mark (') means counter-clockwise, and 2 means one smooth 180-degree half turn. The move engine also supports middle slices M E S and whole-cube rotations x y z when they appear in algorithms and patterns.

Every move passes through one queue, so button presses, keyboard turns, guide algorithms, scrambles, and patterns play in order without overlapping. The large move HUD at the top flashes the token currently being performed.

Keyboard shortcuts

Press U, D, L, R, F, or B for a clockwise face turn; hold Shift for the prime version. Press Escape to close an open TECH or HELP overlay. When no overlay is open, Escape toggles the solving guide.

Scramble, Solve, and Reset

SCRAMBLE performs 22 fast random face turns and never chooses the same face twice in succession. It also clears the visible pattern sequence strip.

SOLVE works from any state created in CUBED—including manual turns, scrambles, guide algorithms, stacked patterns, slice moves, and whole-cube rotations. It cancels moves and patterns that have not yet begun, allows a turn already in motion to finish safely, then animates the exact inverse sequence back to solved. The solution appears in the sequence strip as AUTO SOLVE. Manual turns, Scramble, patterns, and guide Play buttons are temporarily ignored while solving; RESET can stop the process immediately.

RESET clears queued moves, waiting patterns, and solution history; lets any turn already in motion finish safely; rebuilds a perfectly solved cube; and clears the sequence strip. Reset keeps your currently selected color palette.

Pattern gallery and sequence playback

The Pattern Gallery contains 25 classic decorative patterns. A selected pattern begins from the cube’s current state—patterns are deliberately cumulative, and RESET is the only automatic return to solved. Selecting a pattern immediately hides the control deck for a cleaner view; use the CONTROLS rail button to reveal it again.

If a turn, algorithm, scramble, or earlier pattern is still running, the new pattern waits until the queue is completely empty. The dropdown returns to its placeholder after every selection, so the same pattern can be chosen repeatedly. The sequence strip displays the pattern name and every token: the current token is highlighted, finished tokens remain tinted, and upcoming tokens stay muted. The strip remains visible until the next Scramble or Reset.

The Delay slider adds a pause of 0.25–5 seconds after each pattern move except the last. It affects patterns only—not manual turns, the guide, or Scramble—and it is read live, so changing it during playback alters the remaining pauses.

Cube color palettes

The Color Palette dropdown includes the standard Rubik’s scheme plus 20 alternate named palettes. A palette remaps the six color slots—white, yellow, blue, green, red, and orange—across the entire live cube. You can recolor a solved cube, a scramble, a finished pattern, or a cube in motion without changing any cubie’s position or orientation. The chosen palette remains active when RESET is used, but a full page reload returns to the standard palette.

The eight-step beginner guide

GUIDE opens the beginner-solving panel. The header shows the current step and eight progress ticks. Use Back and Next to move through the lessons; on step eight, Start over returns to step one. Every notation block has a ▶ Play button that queues that exact algorithm on the cube from its current state. The panel’s ✕ or the GUIDE rail button hides the guide without changing your cube or lesson position.

Themes

The top theme pills switch among Nebula, Synthwave, Daylight, and Ember. Each theme changes the page background, interface accent, glass panels, scene lighting, rim color, stars, ground shadow, and—where appropriate—the Synthwave grid. Daylight also changes the interface to light glass with dark text. Scene changes blend smoothly rather than switching abruptly.

Rail buttons and clean-view mode

GUIDE shows or hides the tutorial panel. CONTROLS shows or hides the entire move deck; when the deck is hidden, a visible pattern sequence drops toward the bottom edge. Hide both GUIDE and CONTROLS for an uncluttered full-screen cube. Lit rail buttons indicate which panel or overlay is active.

TECH opens an explanation of how the app is built. HELP opens this guide. Only one overlay can be open at a time. Close an overlay with its ✕, by tapping its dimmed backdrop, by pressing its rail button again, or with Escape.

Phones, tablets, and reduced motion

On smaller screens the guide becomes a bottom sheet, the controls wrap into a compact deck, the rail moves to the top, and touch orbit and pinch zoom remain available. The renderer continually checks the visible canvas size so the cube stays centered across orientation changes and in-app browsers. When your device requests reduced motion, cube turns and theme transitions run much faster.
